Part I: Understanding JSON

JSON is a data storage format originally developed for web applications. It has become one of the most popular data exchange formats because of its compact size and readability. (Another popular data format is XML. If you're interested, you can read more about the pros and cons of the two here.

Here is what an array looks like in JSON:

// Array of numbers 
[ 1, 2, 3, 4]

// Array of strings
[ "mon", "tue", "wed", "thu", "fri", "sat", "sun"]

// Array of booleans:
[ true, false, true, true]

JSON stands for "JavaScript Object Notation". Even though we can use it to store simple arrays, it's designed to allow complex objects to be stored in key-value pairs.

The keys are strings that correspond to the property names, and the values are the values of the property for a particular object:

// Player object 
Player p = new Player();
p.setName("Steve");
p.setHealth(20);
p.setMana(50);
p.setCanFly(false);

// JSON Representation 
{ "name": "Steve", "health": 20, "mana": 50, "canFly": false}

We can take an object, serialize into JSON, and then share that data, (either in a file or by sending it over the network. Then we can deserialize it back into an object.

Part II: Use the GSON Library

Unfortunately, Java doesn't have built-in support for JSON, so to parse it we have use a 3rd party library. There are several available, but we'll be using Google's GSON library (GSON just stands for "Google JSON", it isn't a different data format).

We'll be using a pre-compiled jar version of the library. A jar file is like a zip file containing a bunch of compiled java classes. It's how java programs and libraries are typically shared.

  1. To begin, download this sample project.

  2. Read through the comments in the code to understand what is happening.

  3. To compile the code, navigate to the directory containing the java files and execute this command:

    javac -cp "./lib/gson-2.8.0.jar" *.java

    The -cp flag means "Class path", and it allows to tell the compiler where it can find extra code we want to reference when we compile.

  4. Unlike C++, Java doesn't include the library permanently in our code. This means we still need to reference it when we execute the code:

    java -cp ".;./lib/gson-2.8.0.jar" Parser

    In this case, the -cp flag tells the Java runtime everywhere we want it to look for code. We tell it to first look in the current directory (.) and to also look in the GSON jar file in the lib directory (./lib/gson-2.8.0.jar).

    The ; is just a separator between each location. (If you're running java on a mac or linux computer, you'll need to use a : for your path separator rather than ;).
